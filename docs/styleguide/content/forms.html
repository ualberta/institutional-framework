Forms

Form Layout

The standard form layout should have a fieldset with a legend that accurately labels the fields inside the fieldset. Text, textarea, and select fields should have a label above them and placeholder text that describes which type of data should be entered in the field. Radio buttons and checkboxes should have labels on the right side of the field. Success notifications or validation errors should appear to the right of the corresponding field when used.

Example
Sign Up For Our Newsletter
Markup
<form>
   <fieldset>
	  <legend>Sign Up For Our Newsletter</legend>
	  <label>Your Name</label>
	  <input type="text" placeholder="First Last">
	  <label>Email Address</label>
	  <input type="text" placeholder="Enter your email address">
	  <label>Country</label>
	  <select>
	  	<option>Canada</option>
	  	<option>Not Canada</option>
	  </select>
	  <label class="checkbox">
	    <input type="checkbox"> Plain Text Only
	  </label>
   </fieldset>
   <button type="submit" class="btn btn-grey btn-small">Submit</button>
</form>

Inline Forms

If a form only requires one visible field it should be displayed inline with the button. The form should have a heading, and at very least a placeholder describing what type of data to ender in the field. Examples of this type of form include search forms, find a person, or newsletter subscribe forms. Icons may be used to help inform the user the form's purpose.

Example
Markup
<div class="inline-search">
    <i class="icon-search"></i>
    <input type="search">
    <button class="btn btn-grey btn-small">Search</button>
</div>
